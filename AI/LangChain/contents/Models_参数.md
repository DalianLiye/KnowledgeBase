# 关于参数

初始化Chat Model时可通过参数控制模型行为\
不同厂商参数略有差异，通用核心参数如下：

- **model**\
  必填，指定模型名称；可通过“提供商:模型名”格式指定厂商，如 openai:gpt-4o。

- **api_key**\
  厂商身份认证密钥，通过平台申请获取；建议配置在环境变量，不同厂商环境变量名不同。

- **temperature**\
  控制模型输出的随机性；值越高，创意越强；值越低，结果越确定、越严谨

- **max_tokens**\
  限制模型返回的最大token数量，控制输出长度

- **timeout**\
  请求超时时间（秒），超过则自动取消

- **max_retries**\
  请求失败时最大重试次数，默认 6\
  采用带抖动的指数退避策略：等待时间随重试次数指数增长，并加入随机抖动，避免大量请求同时重试造成风暴\
  自动重试：网络错误、429 限流、5xx 服务端错误\
  不重试：401、404 等客户端非法错误\
  网络不稳定场景可设为 10–15

  关于重试使用带抖动的指数退避策略\
  在出现失败（比如网络请求超时、临时错误等）时，会尝试重新发送请求（重试）\
  重试之间的等待时间按照“指数退避”（exponential backoff）增加\
  比如，第一次重试等待短时间，第二次等待大约是第一次的两倍，第三次再翻倍，以此类推，这样可以减少对服务的瞬时压力并增加成功率\
  “带抖动”表示在每次计算的退避等待时间上加入随机扰动（随机化），避免大量客户端在相同退避间隔同时重试，从而防止“重试风暴”或同步冲突\
  抖动可以是把等待时间随机化到 0 到计算出的退避值之间，或者在退避值的上下某个范围内随机选择


示例：
```python
model = init_chat_model(
    "claude-sonnet-4-6",
    # Kwargs passed to the model:
    temperature=0.7,
    timeout=30,
    max_tokens=1000,
    max_retries=6,  # Default; increase for unreliable networks
)
```
