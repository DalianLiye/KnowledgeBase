# ANTLR生成的文件 #
ANTLR解析XXXXX.g4，会生成以下几个文件，以java为例\

- **XXXXXParser.java**\
包含了一个语法分析器类的定义，该语法分析器专门用来识别定义的XXXXX语法的\
该类中，每一个规则都有一个对应的方法，以及一些其他辅助的方法

- **XXXXXLexer.java**\
ANTLR能够自动识别出我们的语法中的文法规则和词法规则\
这个文件包含的是词法分析器的类定义，它是由ANTLR通过分析词法规则，以及语法规则中的字面值(就是用单引号''括起来的字符串)生成的\
词法分析器的作用是将输入字符序列分解成次会符号

- **XXXXX.tokens**\
ANTLR会给每个我们定义的词法符号指定一个数字形式的类型，然后将它们的对应关系存储与该文件中\
有时，我们需要将一个大型语法切分为多个更小的语法，这种情况下，这个文件就非常有用了\
通过它，ANTLR可以在多个小型语法间同步全部的词法符号类型

- **XXXXXListener.java和XXXXXBaseListener.java**\
默认情况下，ANTLR生成的语法分析起能将输入的文本转换为一颗语法分析树\
在遍历语法分析树时，遍历器能够触发一系列事件(回调),并通知我们提供的监听对象\
XXXXXListener接口给出了这些回调方法的定义，我们可以实现它来完成自定义的功能\
XXXXXBaseListener是该接口的默认实现类，为其中的每个方法提供了一个空实现\
XXXXXBaseListener使得我们只需要覆盖那些我们感兴趣的回调方法\
通过-visitor命令行参数，ANTLR也可以为我们生成语法分析树的访问器
